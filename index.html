Хто клієнт а хто сервер?

Клієнт

Клієнт - це звичайні користувачі які підключені до мережі і здійснюють доступ через певне ПО(напр. браузер).

Сервер

Віддалений комп'ютер із постійним(?) стабільним(?) швидким(?) підключенням до мережі, на якому зберігаються сайти чи додатки.

Довга дорога одного запиту

Що робить браузер?

  • Надсилає запит на DNS (8.8.8.8) домен до якого звертаються
  • DNS повертає IP адресу сервера на якому цей домен знаходиться
  • Браузер надсилає HTTP запит на сервер, вказуючи копію якого сайту він хоче отримати
  • Спеціальне ПО (Apache, nginx, etc.) перенаправляють запит саме в ту папку на сервері де лежить потрібний нам сайт

HTTP

HyperText Transfer Protocol

Протокол досі актуальний, хоча був розроблений ще у 90х роках для передачі гіпертексту

HTTPS

HyperText Transfer Protocol Secure

Використання HTTPS забезпечує шифрування всіх заповідей/відповідей між клієнтом та сервером

Поговоримо про REST

Чому? Тому що це найбільш використовуваний підхід (у різних імплементаціях) до комунікації між клієнтом та сервером

REST

Representational State Transfer

Що нам потрібно?

  • HTTP методи
    • OPTIONS - інформаційний
    • GET - Отримати данні
    • POST - Записати данні
    • PUT - Оновити данні
    • DELETE - видалити данні

Що нам потрібно?

  • HTTP статус коди
    • 1** - Інформаційні
    • 2** - Успішні
    • 3** - Перенаправлення
    • 4** - Клієнтська помилка
    • 5** - Серверна помилка

Популярні статус-коди

  • 200 OK
  • 201 Created
  • 301 Moved Permanently
  • 302 Moved Temporarily

Популярні статус-коди
